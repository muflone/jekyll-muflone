Installazione dal codice sorgente

Se non fosse disponibile il pacchetto preassemblato per la propria distribuzione è possibile utilizzare il pacchetto col codice sorgente per installare {{ include.module.name }} utilizzando:

cd "~/cartella con il file scaricato"
mkdir build
tar xzf "nome del file.pkg.tar.gz" -C build
cd build/{{ include.module.lowername }}-*
python2 setup.py build
sudo python2 setup.py install
cd ..

Al termine dell'installazione, se non sono stati riportati errori sarà possibile eliminare la cartella build da root oppure tramite sudo rm -rf "build" e il file sorgente scaricato.

Attenzione! Dopo aver eseguito la procedura di installazione manuale da codice sorgente non sarà possibile disinstallare automaticamente il software ma sarà necessario eliminare manualmente tutti i files installati che sono indicati durante l'ultimo passaggio del processo di installazione.

Installazione dal codice sorgente

Per compilare {{ include.module.name }} dal codice sorgente è necessario avere anche {{ site.data.remmina-plugin-builder.name }}, un ambiente di sviluppo minimale per la compilazione di plugin per Remmina.

Scaricare {{ site.data.remmina-plugin-builder.name }} dalla pagina del progetto e fare riferimento alla pagina Installazione.

Il processo di compilazione richiede semplicemente l'estrazione del codice sorgente sia di {{ site.data.remmina-plugin-builder.name }} sia di {{ include.module.name }} utilizzando:

cd "~/cartella con il file scaricato"
tar xzf remmina-plugin-builder-*.tar.gz
tar xzf {{ include.module.lowername }}-*.tar.gz

Quindi sarà necessario compilare il plugin con:

mkdir build
cp -r {{ site.data.remmina-plugin-builder.lowername }}-*/* build
cp -r {{ include.module.lowername }}-*/* build/remmina-plugin-to-build
cd build
cmake -DCMAKE_INSTALL_PREFIX=/usr .
make
sudo make install
cd ..

Al termine dell'installazione, se non sono stati riportati errori sarà possibile eliminare la cartella build e il file sorgente scaricato.

Attenzione! Dopo aver eseguito la procedura di installazione manuale da codice sorgente non sarà possibile disinstallare automaticamente il software ma sarà necessario eliminare manualmente tutti i files installati che sono indicati durante l'ultimo passaggio del processo di installazione.

Installation from source code

If the prebuilt package will not be available for your distribution you can use the source code package to install {{ include.module.name }} using:

cd "~/folder with the downloaded file"
mkdir build
tar xzf "file name.pkg.tar.gz" -C build
cd build/{{ include.module.lowername }}-*
python2 setup.py build
sudo python2 setup.py install
cd ..

After the installation, if no errors were reported the folder build can be removed from root or via sudo rm -rf "build" and the same for the downloaded source package.

Warning! After executing the manual installation from source code an automatic software uninstall will not be possibile but you will have to manually delete all the installed files that were reported during the last installation step.

Installation from source code

To build {{ include.module.name }} from the source code you will need also {{ site.data.remmina-plugin-builder.name }}, a minimal development environment to build plugins for Remmina.

cp -r {{ site.data.remmina-plugin-builder.lowername }}-*/* build
cp -r {{ include.module.lowername }}-*/* build/remmina-plugin-to-build
cd build
cmake -DCMAKE_INSTALL_PREFIX=/usr .
make
sudo make install
cd ..

After the installation, if no errors were reported the folder build can be removed and the same for the downloaded source package.

Warning! After executing the manual installation from source code an automatic software uninstall will not be possibile but you will have to manually delete all the installed files that were reported during the last installation step.
